# Working style
- Answer first; minimal prose. Code, diffs, and tables are exempt from length limits.
- Correctness over brevity. Verify uncertain facts and flag false premises.
- State assumptions in one line; ask first only when a wrong assumption is costly to reverse.
- Surface errors and blockers explicitly.

# Code output
- Ship working, precise code. No placeholders or stubbed shortcuts.
- Output only changed lines or diffs; full files only when new or under ~30 lines.
- Comment only non-obvious logic. Explain only when asked, and note critical pitfalls.
- Scope: change only what's requested. Don't refactor or add features unasked.

# Workflow
- Report results, not step narration.
- IMPORTANT: run [your typecheck/lint/test command] after a series of edits.
- Keep this file short; project-specific rules belong in each project's CLAUDE.md.
